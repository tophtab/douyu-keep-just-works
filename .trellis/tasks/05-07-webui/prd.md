# 优化 WebUI 列表视觉统一性

## Goal

统一 WebUI 中所有列表组件的视觉设计，解决列间距不一致、表格样式不统一、空状态样式混乱等问题，提升整体视觉一致性和用户体验。

## What I already know

### 当前列表组件清单
- **粉丝牌列表**（概况页）：`.fans-status-table`，8列，显示粉丝牌状态、等级、亲密度、双倍状态
- **鱼吧列表**（鱼吧签到页）：`.yuba-status-table`，7列，显示鱼吧等级、经验、签到状态
- **背包礼物列表**（临期任务页）：`.backpack-table`，8列，显示背包礼物、数量、过期时间、临期状态
- **保活房间列表**（保活任务页）：`.table`（通用类），8-9列，配置保活分配权重/数量
- **双倍房间列表**（双倍任务页）：`.table`（通用类），9列，配置双倍参与勾选和分配
- **临期房间列表**（临期任务页）：`.table`（通用类），8列，配置临期礼物分配
- **日志列表**（运行日志页）：`.log-line`（非表格），卡片式布局

### 已识别的主要问题

#### P0 问题（必须修复）
1. **表格外边距不统一**
   - 有些用 `margin-top:14px`，有些用 `margin-top:16px`
   - 混用内联样式和 CSS 类
   - 位置：`overview-fans-table-wrap`(14px) vs 其他表格(16px)

2. **列间距不一致**
   - 表格 `th/td` 使用 `padding:12px`
   - 日志列表使用 `padding:8px 10px` + `margin-bottom:8px`
   - 不同组件视觉节奏不统一

3. **空状态样式不统一**
   - 有些只有文本：`<div class="empty">文本</div>`
   - 有些带按钮但没有统一包裹类
   - 按钮位置和间距不一致

#### P1 问题（强烈建议）
4. **表格样式类不统一**
   - 粉丝牌/鱼吧/背包有专用类（`.fans-status-table` 等）
   - 保活/双倍/临期只用通用类（`.table`）
   - 导致 `min-width` 约束不一致（860px/820px/980px vs 760px）

5. **列宽控制不一致**
   - 粉丝牌/鱼吧表格有 `<colgroup>` 精确控制列宽
   - 保活/双倍/临期表格没有 `<colgroup>`，列宽完全自动分配
   - 序号列、操作列宽度不稳定

6. **表格内输入框样式不统一**
   - 表格内 `input[type="number"]` 只约束了 `min-width:96px`
   - 缺少统一的高度、内边距、边框圆角
   - 与表格外输入框样式差异大

#### P2 问题（可选优化）
7. **日志列表视觉重量过重**
   - 每个日志行有独立边框和背景
   - 视觉上比表格更"重"，层次感不协调

8. **移动端响应式不一致**
   - 专用表格和通用表格的响应式行为不同

### 技术约束
- 所有样式和 HTML 都在单文件 `src/docker/html.ts` 中（3574 行）
- 使用内联 CSS（`<style>` 标签）+ JavaScript 动态生成 HTML
- 表格通过 JavaScript 函数构建：
  - `buildFansStatusTable()` - 粉丝牌表格
  - `buildYubaStatusTable()` - 鱼吧表格
  - `buildBackpackRowsTable()` - 背包表格
  - `buildSendTable()` - 保活/双倍/临期表格（通用）
- 样式定义位置：行 680-767（表格）、行 767-797（日志）

## Assumptions

- 用户希望保持现有功能不变，只优化视觉一致性
- 不需要重构成独立 CSS 文件，保持单文件架构
- 优化后的样式应该在浅色/深色主题下都表现良好
- 移动端响应式行为需要保持或改进

## Requirements

### 核心需求
1. 统一所有表格容器的外边距为 `16px`
2. 统一表格和日志列表的内边距为 `12px`
3. 为所有空状态定义统一的样式类
4. 为保活/双倍/临期表格添加 `<colgroup>` 控制列宽
5. 统一表格内输入框的样式（高度、内边距、边框圆角）
6. 统一所有表格的 `min-width` 策略

### 质量要求
- 修改后不影响现有功能
- 浅色/深色主题下都正常显示
- 移动端响应式不退化

- 不改变现有的响应式断点（保持 1100px 和 960px）
