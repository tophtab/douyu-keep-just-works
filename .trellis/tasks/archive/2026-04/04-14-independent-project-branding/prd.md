# Rebrand repo as independent project

## Goal
Refresh the repository README so it follows a clearer GitHub-style structure and presents the project as an independently maintained repository.

## Requirements
- Pull the latest upstream README changes before editing.
- Rewrite the README structure instead of making only minimal top-banner changes.
- Keep deployment, configuration, and development instructions accurate to the current repository.
- Preserve attribution, but move it into a lower-profile history/acknowledgement section.

## Acceptance Criteria
- [ ] README opens with a concise project summary and clear maintenance positioning.
- [ ] README is organized with standard sections such as overview, features, screenshots, quick start, configuration, development, and acknowledgement/history.
- [ ] Existing Docker and local development instructions remain accurate after the rewrite.

## Technical Notes
- Use the current `README.md`, `package.json`, and repository file layout as the source of truth.
- Keep Chinese user-facing documentation style consistent with the existing project.
