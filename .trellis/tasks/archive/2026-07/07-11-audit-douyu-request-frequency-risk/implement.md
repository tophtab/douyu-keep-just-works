# 静态审查执行计划

1. 枚举请求入口
   - 用 `rg` 搜索 Axios、WebSocket、斗鱼域名和动态 URL。
   - 将请求实现映射到自动任务、状态查询、登录/恢复和手动入口。

2. 建立调度与调用链
   - 核对默认 cron、当前脱敏配置、启动行为、手动任务路由和 UI 刷新/扫码轮询。
   - 记录同任务锁、跨任务重叠、缓存 TTL、pending 合并与强制刷新行为。

3. 计算频率与突发上界
   - 分别计算当前本地状态、默认启用任务和 `F=10` 典型配置。
   - 对背包端点回退、鱼吧分页/逐组请求、赠送房间/礼物类型、凭证恢复重试和二维码轮询给出公式。
   - 检查 cron 是否存在最小周期限制以及多个任务同秒触发的可能性。

4. 评估风控信号与现有防护
   - 评估固定 User-Agent、设备 ID、请求间隔、并发度、重试条件、错误提前停止和无全局限速的影响。
   - 按账号/Cookie、Passport、IP、设备和接口维度给出风险等级。

5. 检查测试与历史证据
   - 检查合同测试是否保护调度互斥、缓存合并、并发上限、间隔、重试预算和强制刷新请求数量。
   - 吸收历史任务中已经确认的 rate-safety、safeAuth 轮换和缓存策略结论。

6. 产出并校验 `audit.md`
   - 确保每个中高风险项都有 `file:line` 和可复核计算。
   - 明确回答当前本地状态、默认配置和高风险配置三种结论。
   - 运行 `npm run test:contracts` 验证当前合同基线；运行 `git diff --check` 和 `git status --short` 确认无业务代码改动与无格式错误。

## Review Gates

- 开始执行前：用户审核 `prd.md`、`design.md`、`implement.md` 并批准进入执行阶段。
- 报告完成后：按 Trellis 质量检查流程复核证据覆盖、频率公式和最终结论。

## Risky Areas and Rollback Points

- `config/config.json` 含敏感字段：仅通过 `jq` 构造脱敏视图，禁止整文件输出或写入任务工件。
- 不运行会发起真实斗鱼请求的应用、测试或脚本；合同测试必须使用现有 mock/离线路径。
- 如发现合同测试包含真实网络访问，立即停止该验证命令并改用静态检查。
