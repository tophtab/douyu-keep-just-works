# Go 后端重写方案评估

## Decision

终止 Go 后端重写，改为继续使用 TypeScript，并建立面向 AI 维护的架构收敛任务。用户的首要目标是降低修改时需要理解的文件、概念、合同所有者和兼容分支；Go 不会自动降低这些成本，且会失去 Vue 与后端之间的 TypeScript 类型共享，同时增加 DTO、错误处理和跨语言合同维护。

Go 评估没有进入业务实现阶段。原 TypeScript 中间实现仍保存在 `wip/ts-normalize-parameter-names-order@0a963e1`，作为行为和迁移参考。

## Carried Decisions

- 原参数治理中有实际防错收益的配置命名、公开合同排序、配置与运行时类型分离继续保留。
- 顶层配置、登录凭据、任务配置、诊断响应、默认值、示例和 API 序列化采用稳定 canonical 顺序。
- 导入、局部变量、内部临时对象和没有误传风险的函数参数不做机械排序或改名。
- `enabled` 表示配置开关，`active` 只表示运行时状态。
- 持久化分配意图与运行时赠送任务分离。
- 当前 TypeScript 继续使用 npm `cron` 的六字段格式，不做 Linux 五字段迁移。
- 保活默认值改为每周三 08:00（`Asia/Shanghai`）：`0 0 8 * * 3`。
- 旧的保活默认值 `0 0 8 */7 * *` 迁移到新的周三计划；用户自定义的其他六字段表达式保持原有语义。

## Outcome

本评估任务以“拒绝 Go 重写并交接给 TypeScript 收敛任务”为完成条件。后继任务必须先完成范围收敛和合同规划，再进入实现阶段。
